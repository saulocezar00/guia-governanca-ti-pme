# 📊 Guia Prático de Governança de TI e Métricas de Software para PMEs

Repositório desenvolvido para a disciplina de **Governança de TI e Engenharia de Software**, apresentado à **Professora Anelise**. O objetivo deste projeto é desmistificar e simplificar a aplicação de frameworks robustos de governança (ITIL/COBIT) e métricas de dimensionamento de software (APF/COSMIC) para o cenário de pequenas e médias empresas (PMEs).

---

## 🎯 Objetivos do Projeto

1. **Simplificação de Processos:** Adaptar as melhores práticas do COBIT e ITIL para realidades enxutas.
2. **Mensuração de Valor:** Demonstrar como transformar a percepção da TI de "centro de custo" para "investimento estratégico" usando APF e COSMIC.
3. **Cultura Open Source:** Utilizar o GitHub como o *Single Source of Truth* (Fonte Única da Verdade) para documentação e governança viva (*Governance as Code*).

---

## 🗺️ Planejamento Estratégico (5W2H)

Para a execução deste projeto, utilizamos a metodologia **5W2H** para garantir clareza técnica e entregas tangíveis:

* **What (O quê?):** Desenvolvimento de um Guia Prático de Governança e Métricas.
* **Why (Por quê?):** Garantir a perenidade, conformidade legal e alinhamento estratégico das PMEs com a tecnologia.
* **Where (Onde?):** Centralizado neste repositório do GitHub.
* **Who (Quem?):** Equipe de Engenharia de Software.
* **When (Quando?):** Ciclo acadêmico de 2026.
* **How (Como?):** Filtragem de processos do ITIL/COBIT e criação de templates prontos para uso.
* **How Much (Quanto custa?):** Custo zero de software, utilizando ferramentas de código aberto.

---

## 📁 Estrutura do Repositório

* `/.github`: Modelos de Issues padronizados para o gerenciamento de incidentes e melhorias (alinhado ao ITIL).
* `/documentacao`: Guias detalhados da adaptação dos frameworks e uso prático das métricas de software.
* `/templates_governanca`: Modelos de políticas de segurança e planos de contingência prontos para PMEs assinarem e adotarem.

---

## 🛠️ Como Utilizar Este Guia

1. **Fase de Alinhamento (COBIT):** Comece lendo o arquivo `documentacao/02_adaptacao_itil_cobit.md` para entender como alinhar as metas de negócio com a TI.
2. **Fase de Operação (ITIL):** Utilize os templates da pasta `/templates_governanca` para formalizar as regras mínimas de segurança da empresa.
3. **Fase de Engenharia (APF/COSMIC):** Siga o guia em `documentacao/03_guia_metricas_apf_cosmic.md` para estimar o tamanho e o custo real das suas entregas de software.
